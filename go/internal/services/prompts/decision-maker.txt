## Instructions

**Step 1 — Extract from snippets**
For each column, check if the value is directly stated in a snippet. Extract it if so, and add that result's URL to `source_urls`. Skip extraction if the data is about a different entity (a founder, competitor, or subsidiary of {{entity}} — not {{entity}} itself).

Type rules: number → bare numeric (228000), string → text, boolean → true/false, date → YYYY-MM-DD.

**Step 2 — Select URLs to crawl**
For every column you could not extract, select up to {{max_urls}} URLs likely to contain it. Prefer: the official site of {{entity}}, its Wikipedia page, authoritative registries. Avoid SEO aggregators and pages about other entities. Do not return an empty `urls_to_crawl` when relevant URLs exist in the results.

**Step 3 — Confidence scoring**
For every field in `extracted_data`:
- 1.0: explicitly stated, entity is unambiguous
- 0.8–0.9: clearly stated, minor interpretation needed
- 0.6–0.7: partial info or mild entity ambiguity
- 0.4–0.5: approximate value or entity unclear
- <0.4: heavy inference or likely wrong entity

## Examples

Data in snippet → extract it:
Snippet: "Microsoft had 228,000 employees as of June 2025" (en.wikipedia.org/wiki/Microsoft)
→ {"extracted_data": {"employee_count": 228000}, "source_urls": ["https://en.wikipedia.org/wiki/Microsoft"], "urls_to_crawl": []}

Data not in snippets → crawl:
Column: logo_url. Snippets mention anthropic.com and Getty Images photos of Anthropic.
→ {"extracted_data": null, "source_urls": [], "urls_to_crawl": ["https://www.anthropic.com", "https://www.gettyimages.com/..."]}
